# Pre-3.0.0 Verification Gauntlet — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run a staged verification gauntlet over the 19 modified files + 145 new/changed cards + share section + auditor rewrite that are sitting local-uncommitted, so we know exactly which P0/P1 issues to fix before committing the voice batch and bumping `package.json` from `2.1.0` to `3.0.0`.

**Architecture:** Six phases, cheap gates before expensive ones. Phase 0 is a fail-fast pre-flight (build/test/typecheck). Phases 1-3 run domain audits and converge on a single P0/P1/P2 list. Each phase writes a dated report to `.audits/2026-04-28/`. Rework loops fix only what the gate flagged so downstream gates aren't re-run unnecessarily. The plan terminates at "ready to commit" — actual commit/push/version-bump happens only after explicit per-action user approval per `feedback_seedhaplot_no_prod_push.md`.

**Tech Stack:** Vite + React 18 + TypeScript + vitest + Puppeteer for screenshots; agents `seedhaplot-card-auditor`, `seedhaplot-fact-checker`, `seedhaplot-visual-qa`, `seedhaplot-player-simulator`, `seedhaplot-panel`.

## Severity legend

- **P0 — blocker for 3.0.0 ship.** Broken UX, factual contradiction, hard-rule voice violation, build red.
- **P1 — strongly recommended fix.** B-grade voice, layout regression, weak share copy, missing analytics event on a new screen.
- **P2 — defer to roadmap.** Polish, nice-to-have, parked feature.

Phase 4 ships only if P0 = 0 and P1 backlog has been consciously triaged (fix or defer with reason).

The fact-checker should not waste WebFetch budget on cards that haven't changed. Compute the delta first.

```bash
git show 3a50ced:cards.json > /tmp/cards-baseline.json 2>/dev/null && \
  jq -r '.[] | .id' /tmp/cards-baseline.json | sort > /tmp/baseline-ids.txt && \
  jq -r '.[] | .id' cards.json | sort > /tmp/current-ids.txt && \
  comm -13 /tmp/baseline-ids.txt /tmp/current-ids.txt > .audits/2026-04-28/_changed-ids.txt && \
  wc -l .audits/2026-04-28/_changed-ids.txt
```

Expected: ~145 new ids. (Modified-but-same-id cards aren't captured by `comm`; supplement with the next step.)

```bash
jq -r --slurpfile base /tmp/cards-baseline.json '
  . as $cur
  | $base[0] as $bs
  | ($bs | map({(.id): .}) | add) as $bm
  | $cur
    | map(select(.id as $id | $bm[$id] != null and ($bm[$id].c != .c or $bm[$id].f != .f or $bm[$id].n != .n)))
    | .[] | .id
' cards.json | sort >> .audits/2026-04-28/_changed-ids.txt && \
  sort -u -o .audits/2026-04-28/_changed-ids.txt .audits/2026-04-28/_changed-ids.txt && \
```
